Why MINTverse?

Traditional malaria simulation models can take hours or days to run complex scenarios. MINTverse provides instant predictions using pre-trained machine learning models, enabling rapid intervention planning and what-if analysis.

Quick Start

Get predictions in seconds with just one function call 

# Install MINTer (automatically installs all dependencies)
                        devtools::install_github("CosmoNaught/MINTer")

                        # Core functionality
                        library(MINTer)
                        library(dplyr)   # Pretty console tables
                        
                        results <- run_mint_scenarios(
                          # —— Net combination settings ——
                          res_use      = c(0.30, 0.45, 0.20),
                          py_only      = c(0.40, 0.30, 0.50),
                          py_pbo       = c(0.10, 0.15, 0.05),
                          py_pyrrole   = c(0.00, 0.05, 0.00),
                          py_ppf       = c(0.10, 0.10, 0.15),
                        
                          # —— Malaria-environment settings ——
                          prev_vec       = c(0.20, 0.40, 0.60),
                          Q0_vec         = c(0.65, 0.75, 0.85),
                          phi_vec        = c(0.45, 0.65, 0.75),
                          season_vec     = c(0, 1, 1),
                          routine_vec    = c(0, 0, 1),
                          irs_vec        = c(0.10, 0.35, 0.70),
                          irs_future_vec = c(0.20, 0.40, 0.50),
                          lsm_vec        = c(0.05, 0.45, 0.85),
                          year_start     = 2,
                          year_end       = 5
                        )
                        
                        # ——— Save CSVs ———
                        write.csv(results$prevalence, "results_prevalence.csv", row.names = FALSE)
                        write.csv(results$cases,       "results_cases.csv",       row.names = FALSE)
                        
                        # Pretty cases table
                        results$cases |>
                          transmute(year,
                                    cases = ensemble_cases_per_1000,
                                    scenario)
                        
                        # Prevalence plots
                        create_scenario_plots(results$prevalence, output_dir = "plots")

MINTer

Malaria INTervention emulator in R

View on GitHub

MINTer is the main orchestration package that brings together the entire MINTverse ecosystem. It provides a simple, unified interface for running complex malaria intervention scenarios.

✨ Direct Emulation

Predict prevalence or clinical cases in seconds using pre-trained neural networks

🔄 Full Pipeline Mode

Run malariasimulation, store in DuckDB, then emulate for benchmarking

🐍 Python-Free

Python/Torch configured automatically via reticulate - no manual setup

🔗 Integration

Seamlessly integrates estiMINT for prevalence→EIR conversion

Key Functions

Function Purpose Returns
run_mint_scenarios() Main function - runs complete analysis pipeline List with prevalence and case predictions
run_malaria_emulator() Direct neural network predictions Data frame with time series predictions
create_scenarios() Create intervention scenario specifications Scenario data frame
create_scenario_plots() Generate publication-ready visualizations ggplot2 figures saved to disk

estiMINT

Machine learning models for EIR estimation and case prediction

View on GitHub

estiMINT provides fast, transparent machine learning models for converting field prevalence data into entomological inoculation rates (EIR) and predicting annual case incidence.

🎯 EIR Estimation

Convert prevalence + coverage data into initial EIR values

📊 Case Prediction

Project annual cases per 1000 population for years 3-6

🚀 Pre-trained Models

Ships with validated XGBoost and Random Forest models

🔧 Customizable

Train your own models on simulation outputs

Model Performance

Validation Metrics (held-out test data):
• EIR models: RMSE < 15 bites/person/year (R² > 0.95)
• Case models: RMSE < 25 cases/1000/year (R² > 0.95)

Example Usage

# Load pre-trained models
pretrained <- load_pretrained_eir_models()

# Estimate EIR from field data
field_data <- data.frame(
  prevalence = 0.35,
  itn_use    = 0.40,
  irs_use    = 0.15
  # ... other parameters
)

eir_estimate <- predict_initial_eir(
  pretrained$ensemble, 
  field_data, 
  pretrained$feature_cols
)

MINTe

Neural network time series forecasting

View on GitHub

MINTe is a Python package providing RNN models (GRU and LSTM) for time series forecasting of malaria prevalence. It features automatic hyperparameter tuning and GPU acceleration.

🧠 Neural Networks

LSTM and GRU architectures optimized for malaria time series

⚡ GPU Support

CUDA acceleration for fast training and inference

🔬 Hyperparameter Tuning

Automatic optimization with Optuna

📈 Rich Metrics

Comprehensive evaluation with R², RMSE, MAE

Note: MINTe is automatically called by MINTer when needed. Python dependencies are installed automatically via reticulate on first use.

Architecture Details

Component Description
Input Features 30-day rolling windows of intervention parameters
LSTM Layers 2 layers, 128-256 hidden units, dropout 0.2
GRU Layers 2 layers, 128-256 hidden units, dropout 0.2
Training Adam optimizer, early stopping, learning rate scheduling
